# v0.0.1 — 2026-03-15

(D) 2026-03-15

### Added

- Notification system: admin can create notifications that are delivered to targeted roles/users.
- Notification bell UI in global header with unread count badge.
- Admin notification management page (create / delete notifications).
- API support for fetching user-specific notifications and marking them as read.

### Changed

- Updated notification API to only return unread items for regular users.
- Notification schema updated to track read status per user.
- Header now renders `NotificationBell` for authenticated users.

### Fixed

- Ensured notification reads are persisted and UI updates after marking read.

### Security (if applicable)

- Added authorization checks so only admins can create/delete notifications.

### Internal

- Added `readBy` field to notification model for tracking per-user reads.
- Updated API endpoints for notification read state management.
